# Primeiro trecho

O engenheiro de Software é o engenheiro mais "teórico que existe", ele trabalha com a parte menos tangível da matéria comparado aos outros engenheiros que costumam trabalhar pensando precisamente na parte física sólida da matéria, normalmente o engenheiro de software é relacionado ao estudo de programação e linguagem. Com isso, essa área de estudo se torna mais diferente ainda quando se observa a parte de linguagem, no qual, é necessário conhecer a prender praticamente uma nova gramática quando se tem como objetivo aprender uma nova linguagem de programação, transparecendo assim uma área menos tangível ainda, formado por ideias que funcionam na prática.

#Segundo trecho

Essa área de estudo é como qualquer outra, tem ferramentas necessárias para usar na profissão, e essas ferramentas se encaixam melhor em ocasiões diferentes, as próprias linguagens de programação reagem a esse tipo de ideia, Python pode ser melhor para algumas aplicações, e C++ melhor para outras, depende da complexidade, tempo e escala do projeto.

#Trade offs

Carro(Segurança), é mais seguro por conta da sua carcaça física x moto(Preço), é mais barato para manter, porém não é mais seguro

JavaScript(Simplicidade), é a própria linguagem de programação x React(Foco em interfaces), é mais complexa, porém tem um escopo melhor definido para o desenvolvimento

Energia Solar(Mais sustentável para a natureza), ecologicamente correta x Energia Distribuida pela rede elétrica(Mais acessível), é mais barata e acessível para camadas menos favorecidas financeiramente da sociedade
